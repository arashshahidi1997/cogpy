# ECoG Signal Characterization - New Measures Brainstorm

## Overview

Brainstorm for new diagnostic measures targeting two classes of ECoG signal contamination, plus broader characterization gaps identified from a codebase audit of `cogpy.core`.

Context: cogpy already has a rich feature registry (34 implemented measures across temporal, spectral, spatial, and bivariate domains — see `pixecog/docs/reference/codelib/feature-registry/`). The proposals below target gaps not covered by the existing registry, with emphasis on **pathological spatial patterns** and **transient narrowband contamination**.

---

## A. Spatial Discontinuity / Striping Detection

Problem: Striped high-power artifacts (checkerboard or column-wise anomalies) are non-physiological but not caught by existing per-channel temporal measures or isotropic spatial measures like Moran's I.

### A1. Spatial frequency analysis — `spatial_frequency_ratio`

**What**: 2D FFT of each time-frame's grid snapshot → power in spatial frequency space.

**Why**: Striping concentrates energy along one spatial-frequency axis (column-wise artifact = energy along ML spatial frequency). A balanced ratio ≈ 1 is normal; extreme values flag striping.

**Where**: `cogpy.measures.spatial`

**Depends on**: existing grid data model (AP, ML dims), numpy 2D FFT.

**Approach**: For each time frame, compute `np.fft.fft2` of the (AP, ML) grid, sum power along the AP spatial-frequency axis vs the ML axis, return ratio.

### A2. Directional Moran's I — extend `moran_i`

**What**: Current `moran_i` uses queen/rook adjacency (isotropic). Add **AP-only** and **ML-only** neighbor variants.

**Why**: Checkerboard = negative I isotropically. Column-stripe = negative along ML but positive along AP. Directional decomposition distinguishes these.

**Where**: extend `cogpy.measures.spatial.moran_i` with `adjacency="ap_only" | "ml_only"`.

**Effort**: Low — just filter the weight matrix by axis before computing the same statistic.

### A3. Spatial gradient anisotropy — `gradient_anisotropy`

**What**: Compute `|∂V/∂AP|` vs `|∂V/∂ML|` per frame → ratio measures directional smoothness.

**Why**: Striped artifacts have very high gradient in one axis but low in the other. Isotropic measures (CSD, Moran's I) average this away.

**Where**: `cogpy.measures.spatial`

**Approach**: `np.gradient` along each spatial axis, then ratio of mean absolute gradients.

### A4. Row/column energy profile — `marginal_energy_outlier`

**What**: Per frame, marginalize power over AP → get ML energy profile (and vice versa). Flag outlier columns/rows via robust z-score.

**Why**: Directly tells you *which* column is bad. Simplest, most interpretable spatial artifact detector.

**Where**: `cogpy.measures.spatial`

**Approach**: Sum of squares along one grid axis, robust z-score across the other axis, threshold.

### A5. Windowed spatial kurtosis

**What**: Kurtosis of the spatial power distribution at each time window.

**Why**: High spatial kurtosis = power concentrated in few electrodes = focal artifact. Low = distributed (normal or common-mode).

**Where**: `cogpy.measures.spatial`

## B. Transient Line Noise Detection (Unknown Frequencies)

Problem: Standard line noise removal targets known harmonics (60/120/180 Hz). Transient narrowband components at unknown frequencies go undetected. The 40 Hz TTL artifact (carrier + 10 Hz AM envelope) on column 10 is a known example, but the general case needs frequency-agnostic detection.

### B1. Sliding F-test line scanner — `ftest_line_scan`

**What**: Wrap existing `mtm_fstat` to scan **all frequency bins** in each time window. At each (time_window, freq_bin): compute F-stat → threshold → binary mask of "active lines".

**Why**: This is the principled multitaper approach to "detect line noise across unknown frequencies without a priori assumptions." The F-test specifically detects narrowband sinusoidal components against a broadband background.

**Where**: new function in `cogpy.spectral.features` or `cogpy.detect`

**Output**: `(time_win, freq)` boolean mask or F-statistic map. Can feed into `EventDetector` framework.

**Priority**: HIGH — directly answers the core question in the idea doc.

### B2. Spectral peak tracker — `track_spectral_peaks`

**What**: In each windowed PSD, find peaks via `scipy.signal.find_peaks` with prominence thresholds. Classify as: known harmonic (60/120/180), known artifact (40 Hz TTL), or **unknown narrowband**. Track peak persistence across time.

**Why**: Simpler than F-test; gives interpretable peak frequency + prominence. Intermittent peaks = noise bouts.

**Where**: `cogpy.spectral.features`

### B3. Narrowband-to-broadband ratio — `narrowband_ratio`

**What**: For each freq bin, ratio of power to local spectral neighborhood (e.g., ±5 Hz flanking median). Time-resolve this to detect when lines turn on/off.

**Why**: Simplest alternative to the F-test scanner. No multitaper machinery needed — works on any PSD estimate.

**Where**: `cogpy.spectral.features`

### B4. Generalized AM envelope detector — extend `am_artifact_score`

**What**: Generalize beyond a single (fc, fm) pair: scan for sideband structure across carrier/modulation frequency space. Output: set of detected (carrier, modulator) pairs + scores.

**Why**: The 40 Hz + 10 Hz TTL artifact is one instance. Other AM artifacts may exist at unknown carrier/modulation frequencies. A scan mode would find them.

**Where**: extend `cogpy.spectral.features.am_artifact_score`

## C. Joint Spatial-Spectral Characterization

### C1. Spatially-resolved line noise map

**What**: For each electrode, compute the line-scanner (B1) or narrowband ratio (B3) over time. Cross-reference with spatial position to detect if noise is localized to specific columns/rows.

**Metric**: `spatial_noise_concentration` — fraction of total noise power attributable to a single row/column.

**Why**: The TTL artifact localizes to column 10 on 16×16 grids. Automated spatial localization separates trivial trimming from distributed noise problems.

### C2. LineNoiseBoutDetector — `cogpy.detect`

**What**: New `EventDetector` subclass that:
1. Computes windowed F-test or narrowband ratio across all frequencies
2. Thresholds to get binary activation mask
3. Finds temporal runs (reuse existing `find_true_runs` + `merge_intervals`)
4. Returns `EventCatalog` with columns: `t0, t1, freq, channel/AP/ML, score`

**Why**: Integrates noise bout detection into the existing `DetectionPipeline` framework. Noise bouts land in `EventCatalog` alongside ripples, bursts, etc.

**Fits**: the unified `EventDetector` ABC, `Transform` pipeline, serialization.

## D. Other Characterization Measures

### D1. Spectral flatness (Wiener entropy)

`geometric_mean(PSD) / arithmetic_mean(PSD)` — 1.0 = white noise, 0.0 = pure tone. Complements spectral entropy; more robust to bandwidth. Simple to add to `cogpy.spectral.features`.

### D2. Inter-channel phase clustering (ICPC)

At each (time, freq): circular variance of phase across channels. High ICPC at a specific freq = volume-conducted or artifactual (shared source); low ICPC = independent neural activity. Could be windowed for temporal tracking.

**Where**: `cogpy.spectral.bivariate` or new `cogpy.measures.spatial`

### D3. Temporal stability index

Autocorrelation of PSD across consecutive windows. Stable PSD = stationary signal; sudden drops = transient contamination epochs. Good for segmenting clean vs contaminated periods.

### D4. Spatial smoothness spectrum

Per-frequency Moran's I: for each freq bin in a windowed spectrogram, compute Moran's I across the grid. Biological signal shows high spatial smoothness at low frequencies, decreasing at high frequencies. Artifacts show anomalous spatial smoothness at specific frequencies.

**Where**: combines `cogpy.measures.spatial.moran_i` with `cogpy.spectral.specx.spectrogramx`

### Implemented measures — summary

**Spectral** (`cogpy.spectral.features`):

- `ftest_line_scan(signal, fs, NW=4.0, p_threshold=0.05)` → `(fstat, freqs, sig_mask)`
  Thomson (1982) eigenvalue-weighted F-test across all freq bins. Detects unknown-frequency sinusoidal lines. 32 tests passing.

- `narrowband_ratio(psd, freqs, flank_hz=5.0)` → `(..., freq)`
  Per-bin power / median(flanking ±bw). PSD-first, batch-compatible. Simpler alternative to F-test.

- `spectral_flatness(psd, freqs)` → `(...,)`
  Wiener entropy: geometric/arithmetic mean ratio. 1=white, 0=tone.

**Spatial** (`cogpy.measures.spatial`):

- `marginal_energy_outlier(grid, robust=True, threshold=3.0)` → dict
  Row/column energy z-scores + outlier flags. Directly identifies striped artifact columns/rows.

- `gradient_anisotropy(grid)` → float
  log2(mean|dV/dAP| / mean|dV/dML|). 0=isotropic, positive=column-stripe, negative=row-stripe.

- `moran_i(grid, adjacency="ap_only"|"ml_only")` — extended
  Directional spatial autocorrelation. Discriminates stripe axis from checkerboard.

## Existing Primitives to Leverage

These already exist in cogpy and should be reused, not reimplemented:

- `mtm_fstat` — F-test for sinusoidal lines (B1 builds on this)
- `moran_i` — spatial autocorrelation (A2 extends this)
- `am_artifact_score` / `am_depth` — AM artifact detection (B4 extends these)
- `line_noise_ratio` — known-freq line detection
- `spatial_coherence_profile` — distance-dependent coherence
- `csd_power` — 2D Laplacian sharpening
- `GridNeighborhood` — full grid adjacency infrastructure
- `find_true_runs` / `merge_intervals` — temporal event extraction (C2 uses these)
- `EventDetector` ABC + `DetectionPipeline` — framework for C2
- `spectrogramx` — multitaper time-frequency (many proposals need this)
- `BurstDetector` / h-maxima — spectral blob detection

## Notes

- All new measures should follow cogpy's two-layer design: pure compute in `cogpy.core`, no I/O
- Return numpy arrays; schema coercion applied at pipeline boundaries
- Follow the PSD-first convention for spectral features (see `pixecog/docs/reference/codelib/feature-registry/design-spectral.md`)
- Consider windowed variants from the start — most of these are useful as time-resolved diagnostics
