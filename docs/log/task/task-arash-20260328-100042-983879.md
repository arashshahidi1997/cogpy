# cogpy.wave travelling-wave subpackage — implementation prompt

## Goal

Extend the existing `cogpy.wave` subpackage with travelling-wave detection and analysis methods for ECoG/MEA grid data. The existing `cogpy.wave` contains waveform morphology tools (extrema detection, peak extraction, wave clustering); the new code adds **spatial propagation analysis** as a sibling layer.

## Context

### Design spec
`docs/reference/research/travelling-waves.pdf` — comprehensive survey of methods, proposed module layout, and Phase 1/2 roadmap. This is the primary blueprint.

### Existing cogpy.wave (preserve as-is)
- `detect.py` — `Extrema` class: local min/max detection on grids, spatial wave propagation via morphological clustering
- `features.py` — `positive_boundaries`, `duration`, `trajectory`, stub feature functions
- `process.py` — `extract_wave_df`, `positive_waves`, `process_wave`: peak-based waveform extraction to DataFrames
- `plot.py` — `shade_waves`: matplotlib fill_between helper
- `utils.py` — `wave_gen`: iterator over wave segments

These are **waveform morphology** tools (1D peak/trough analysis). The new travelling-wave code is complementary — it estimates **direction, speed, wavelength** of spatially propagating patterns.

### Reference codio toolboxes (MATLAB, cloned in shared pool)
All are `reference_only` — port algorithms to Python, don't wrap.

Key method papers — see survey PDF pages 8-11 for full list. The core algorithmic references:
- zhang_2018 (PGD + plane-wave fit blueprint)

### cogpy conventions to follow
- **Data model**: `xarray.DataArray` with dims from `cogpy.base.ECoGSchema` — `"time"`, `"AP"`, `"ML"` for grids; `"ch"` for flat channels. Sampling rate via `sig.fs`.
- **Lazy loading**: `lazy_loader.attach()` in `__init__.py` (see `spectral/`, `measures/` for pattern)
- **Compute vs I/O**: pure compute, no file I/O in this subpackage
- **Deps**: NumPy/SciPy/xarray are core. `scikit-image` is optional (optical flow). Use `cogpy.utils.imports.import_optional()` for optional deps.
- **Tests**: mirror source structure under `tests/wave/`
- **Docstrings**: NumPy style

### Phase 1 — module specs

**`_types.py`** — Core data containers
- `Geometry`: grid spacing (dx, dy) for regular grids OR per-channel (x, y) coords for irregular layouts
- `PatternType`: enum — `planar`, `rotating`, `spiral`, `source`, `sink`, `mixed`, `uncertain`
- `WaveEstimate`: dataclass with fields: `direction` (radians), `speed` (units/s), `frequency` (Hz), `wavenumber`, `wavelength`, `pattern_type`, `confidence`, `fit_quality` (PGD-like), `support_mask`

**`phase_gradient.py`** — Phase-gradient plane-wave fitting (Zhang 2018 style)
- Port algorithm logic from `wm_travelingwaves_code` repo
- `hilbert_phase(data, axis="time")` -> analytic phase via scipy Hilbert
- `phase_gradient(phase, geometry)` -> spatial gradient on grid (finite diff) or scattered points (linear regression)
- `plane_wave_fit(phase, geometry)` -> direction, speed, fit residuals via circular-linear regression
- `pgd(phase, geometry)` -> phase-gradient directionality score (0-1 robustness metric)
- All functions accept `xarray.DataArray` with `(time, AP, ML)` or `(time, ch)` + Geometry
- Return `WaveEstimate`

**`kw_spectrum.py`** — k-w spectral estimation
- `kw_spectrum_3d(data, geometry, *, nperseg, noverlap)` -> 3D FFT -> `xarray.DataArray` with dims `(kx, ky, freq)`
- `kw_peaks(spectrum, *, n_peaks)` -> extract dominant peaks -> list of `WaveEstimate`
- `kw_ridge(spectrum)` -> trace dispersion ridges
- Window-based STFT variant for nonstationarity

**`optical_flow.py`** — Velocity field estimation
- Reference: `neuropatttoolbox` and `ofamm` repos for algorithm logic
- `compute_flow(frames, method="tvl1")` -> velocity field `(u, v)` as DataArray with `(time, AP, ML)`
- Methods: wrap `skimage.registration.optical_flow_tvl1` and `optical_flow_ilk` (optional dep via `import_optional`)
- Input: amplitude maps, phase maps, or complex analytic-signal components

**`vector_field.py`** — Velocity field features
- Reference: `neuropatttoolbox` critical point analysis
- `divergence(u, v, geometry)` -> scalar field
- `curl(u, v, geometry)` -> scalar field (2D curl = vorticity)
- `critical_points(u, v, geometry)` -> locations + classification (source/sink/spiral/saddle)
- `classify_pattern(u, v, geometry)` -> `PatternType`

**`beamforming.py`** — f-k scanning for irregular arrays
- `fk_spectrum(data, coords, freqs, slowness_grid)` -> beam power over slowness vectors
- `capon_beamformer(csd, coords, freqs, slowness_grid)` -> MVDR/Capon high-resolution f-k
- Accept cross-spectral density matrix from `cogpy.spectral`

**`multitaper_nd.py`** — Multidimensional tapers
- `dpss_nd(n_space, n_time, bw_space, bw_time)` -> separable outer-product DPSS tapers
- Integrate with `kw_spectrum.py` as an enhanced windowing mode
- Reference: Hanssen 1997

**`synthetic.py`** — Synthetic wave generator
- `plane_wave(shape, geometry, direction, speed, frequency, noise_std)` -> DataArray
- `spiral_wave(shape, geometry, center, rotation_speed, frequency, noise_std)` -> DataArray
- `multi_wave(components, noise_std)` -> superposition
- `wave_packet(...)` -> spatially/temporally localized

**`surrogates.py`** — Null-model testing
- Reference: `travelling_waves_or_sequentially_activated_discrete_modules` repo
- `phase_randomize(data)` -> preserve amplitude spectrum, randomize phase
- `spatial_shuffle(data)` -> permute channel/spatial positions
- `time_shuffle(data)` -> permute time blocks
- `surrogate_test(data, estimator, n_surrogates, statistic)` -> p-value

**`generalized_phase.py`** — Broadband phase stabilization
- Port from `/storage/share/codelib/mullerlab--generalized-phase`
- `generalized_phase(data)` -> stabilized instantaneous phase for wideband signals
- Center analytic signal, correct negative-frequency components

### Gaps (not covered by existing codio repos)
1. **k-w spectral estimation**: No reference MATLAB code in codio — implement from first principles (3D FFT + peak extraction). Well-defined mathematically.
2. **Capon/MVDR beamforming**: Classical array processing algorithm, not in any cloned repo. Implement from Capon 1969 equations directly.
3. **Multidimensional multitaper**: Hanssen 1997 gives explicit separable construction. No existing code — build from SciPy's `dpss()`.
4. **Phase singularity / spiral detection** (Phase 2): winding number algorithms exist in cardiac literature but not in cloned repos. Defer.
5. **State-space / Kalman tracking** (Phase 2): no reference code. Defer.

### Dependency strategy
- **Core** (always available): numpy, scipy, xarray — sufficient for phase_gradient, kw_spectrum, beamforming, multitaper_nd, synthetic, surrogates, generalized_phase, _types
- **Optional**: `scikit-image` for optical_flow.py — guard with `import_optional("skimage", pip_name="scikit-image")`
- **Existing cogpy**: use `cogpy.spectral` for CSD computation in beamforming; use `cogpy.preprocess.filtering.temporal` for bandpass before phase extraction

### Testing approach
- Every estimator tested against synthetic ground truth: generate known plane wave -> recover direction/speed within tolerance
- Surrogate tests: verify that estimator produces non-significant results on phase-randomized data
- Small grid edge cases: 4x4, 8x8 grids with limited k-resolution
- Irregular geometry: scattered (x,y) coords for beamforming path
