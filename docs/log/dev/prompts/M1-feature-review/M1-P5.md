# CogPy Feature Architecture Assessment

Summarize what already exists vs missing.

## Schema compatibility

## Practical composition assessment
Assess how easily these measures integrate into real project workflows, based on the example Snakemake pipeline.

## Internal pipeline stance
Evaluate whether CogPy should:
- avoid owning workflow orchestration
- expose lightweight composition helpers only
- or introduce any minimal detector/composition abstractions

Be explicit and conservative.

## Proposed APIs

- input schema
- output schema
- vectorization strategy
- xarray compatibility
- external orchestration compatibility

## Schema adjustments

## Composition helpers
Recommend any small reusable helpers that would reduce repeated script glue without turning CogPy into a workflow engine.
