## Cross-correlogram + PETH functions needed for pixecog coupling analysis

pixecog analysis plan §4.1-4.2 needs cross-correlograms and peri-event time histograms. cogpy has no implementation; elephant library (code/lib/elephant) has reference implementations.

### What to study

1. **elephant.spike_train_correlation** — cross-correlogram functions for spike/event trains
2. **elephant.statistics** — time_histogram, peri-event analysis

### What to implement in cogpy

Add to `cogpy.core.measures` (or a new `cogpy.core.coupling` submodule):

1. `cross_correlogram(events_a, events_b, bin_size, window, normalize=True)` — cross-correlogram of two event trains (timestamps or EventCatalog). Returns time-lag bins and counts/rate. Should support both spike trains and LFP event trains (spindle onsets, SWR onsets, DOWN-state onsets).

2. `peri_event_histogram(spikes, events, bin_size, window, baseline=None)` — PETH of spike times aligned to event timestamps. Returns binned firing rate with optional baseline normalization.

3. `spectral_power_xcorr(power_a, power_b, max_lag)` — cross-correlation of two spectral power time series (e.g., ripple power vs spindle power). Log-transform before correlating. Returns lag axis and correlation values.

### Design constraints

- Pure functions: array/xarray in, array/xarray out. No file I/O, no BIDS paths.
- Should accept EventCatalog.df timestamps or raw numpy arrays.
- Surrogate support: accept a `n_surrogates` parameter that runs circular-shift controls and returns observed + surrogate distribution.
- Multi-timescale: should work with both fine (1-5 ms bins, ±500 ms) and coarse (100 ms bins, ±5 s) settings.

### References

- @sirota_2003_CommunicationNeocortex: spectral power cross-correlation method
- @wierzynski_2009_StateDependentSpikeTiming: spike cross-covariance, SWR-conditioned analysis
- @peyrache_2011_InhibitionRecruitment: multi-timescale cross-correlograms
- @siapas_1998_CoordinatedInteractions: ripple-spindle event-train cross-correlation

### Priority

High — these are priority 1-2 measures in the pixecog analysis plan, needed for H1, H2, H4, H5, H6.
