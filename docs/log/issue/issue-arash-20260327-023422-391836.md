## Slow-wave / UP-DOWN state detector needed for pixecog event detection

pixecog analysis plan §2.5 needs cortical UP/DOWN state detection from ECoG LFP. cogpy has no slow-wave detector; yasa and neurodsp libraries (code/lib/) have reference implementations.

### What to study

1. **yasa.detection.sw_detect** — slow wave detection via negative half-wave approach: bandpass 0.3-1.5 Hz, detect negative peaks, find surrounding zero-crossings, apply duration (0.3-1.5 s) and amplitude criteria, compute phase. Returns DataFrame with Start, NegPeak, MidCrossing, PosPeak, End, Duration, ValNegPeak, ValPosPeak, PTP, Slope, Frequency.
2. **neurodsp.burst.dualthresh** — dual-threshold burst detection on band-filtered signals, similar pattern to existing cogpy RippleDetector but for low frequencies.

### What to implement in cogpy

Add `cogpy.core.detect.slowwave`:

1. `SlowWaveDetector(EventDetector)` — following the same pattern as RippleDetector/SpindleDetector:
   - Bandpass delta (0.5-4 Hz) or slow oscillation (0.5-2 Hz)
   - Detect negative peaks (DOWN state troughs) via zero-crossing + trough finding
   - Duration gate: DOWN >80 ms, full cycle 0.3-1.5 s
   - Amplitude threshold: peak-to-trough > mean + k*SD
   - Output: EventCatalog with (onset, trough, midcrossing, peak, offset, duration, amplitude, state)
   - State labeling: UP (between trough and next trough), DOWN (trough neighborhood)

2. `gamma_envelope_validator(lfp, gamma_band=(30, 140), smooth_ms=50)` — compute broadband gamma envelope as UP-state proxy. DOWN troughs should coincide with gamma minima. Returns boolean mask or validation score per detected event.

### Design constraints

- Pure functions: xarray DataArray in, EventCatalog out. Same pattern as RippleDetector.
- Should work on both single-channel and multi-channel (per-channel detection).
- The primary detection for pixecog is blob-on-LFP (h-maxima on delta-filtered signal). This envelope-based detector serves as validation/comparison method. Both should be available.
- Gamma cross-validation is specific to this use case but general enough for cogpy — any cortical UP/DOWN study would want it.

### References

- @isomura_2006_IntegrationSegregation: DOWN = 200-500 ms silence, UP = 0.3-1 s activity, gamma tracks UP
- @swanson_2025_TopographyPutative: 25th percentile binarization, DOWN >80 ms
- @battaglia_2004_HippocampalSharp: delta 0.8-4 Hz, SPW coincides with D-U transitions

### Priority

High — needed for pixecog §2.5, blocks H1, H4, H6.
