# site-build uses rag env which lacks sphinx — add sirotalab profile with cogpy/labpy env

## Summary
- `make site-build` invokes `projio site build` which runs under the `rag` env (`/storage/share/python/environments/Anaconda3/envs/rag/bin/python`). That env does not have `sphinx` installed, so the build fails with `No module named sphinx`.

## Context
- Sphinx is available in the `cogpy` and `labpy` conda environments on sirotalab machines.
- The `rag` env is the projio default but is not appropriate for projects that need Sphinx for doc builds.
- This affects any sirotalab project using `projio site build` with Sphinx-based docs.

## Proposed fix
- Add a sirotalab profile (or per-project override) to projio that selects `cogpy` or `labpy` as the Python environment for site builds.
- Alternatively, allow `projio.toml` or `.projio/config.toml` to specify which conda env to use for `site build`.

- [ ] Document how to configure per-project Python env for site builds
