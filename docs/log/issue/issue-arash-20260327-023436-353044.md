## Enrich SpindleDetector with yasa-style validation features

cogpy has a basic `SpindleDetector` (subclass of RippleDetector with different freq/duration defaults). pixecog analysis plan §2.4 uses blob detection as primary but needs envelope thresholding for validation. The existing SpindleDetector should be enriched with features from yasa.

### What to study

1. **yasa.detection.spindles_detect** — features beyond basic envelope thresholding:
   - Relative sigma power criterion (sigma power / total power in window)
   - Per-event oscillation frequency (peak frequency within spindle)
   - Per-event symmetry index (where in the spindle the peak amplitude occurs; 0.5 = symmetric)
   - Isolation criterion (minimum distance between spindles)
   - Returns rich DataFrame: Start, Peak, End, Duration, Amplitude, RMS, AbsPower, RelPower, Frequency, Oscillations, Symmetry

### What to implement in cogpy

Enrich `cogpy.core.detect.ripple.SpindleDetector`:

1. **Per-event frequency estimation** — after detecting spindle intervals, compute peak frequency within each event (via zero-crossing count or short FFT). Add `frequency` column to output EventCatalog.

2. **Relative sigma power** — compute ratio of sigma-band power to broadband power within each detected event. Add `rel_power` column. Optionally use as a secondary detection criterion (events below threshold are rejected).

3. **Symmetry index** — position of peak amplitude within the spindle duration (0-1 scale, 0.5 = symmetric waxing-waning). Add `symmetry` column.

4. **Isolation criterion** — minimum separation between consecutive spindle onsets (default 0.5-1.0 s). Merge or reject events that are too close.

### Design constraints

- These are enrichment features on detected events, not changes to the core detection logic.
- Should be optional (off by default) to keep the base detector lightweight.
- Output should extend EventCatalog with additional columns, not change existing columns.
- The enriched SpindleDetector is the validation method; blob detection (h-maxima on spectrogram) remains the primary pixecog approach.

### References

- @warby_2014_SleepspindleDetection: frequency 11-16 Hz via FIR + zero-crossing, symmetry score, duration 0.5-3.0 s
- @dimitrov_2021_SleepSpindles: TF-sigma peaks have prominence, duration, central frequency, bandwidth

### Priority

Medium — enrichment of existing detector, not a new capability. Needed for pixecog §2.4 validation.
