# Codex Prompt Reference

This document lists the available Codex prompts, where to find them, and when to
use each one. All prompts rely on the shared path settings in
`prompt/prompt-paths.yaml`.

## Datalad Save

- **Guide:** `prompt/codex-datalad-guide.md`
- **Prompt:** `prompt/datalad.prompt.md`
- **Use for:** Running `datalad save` with a message in the standard conda environment.

## Formatting & Lint

- **Guide:** `prompt/codex-formatting-guide.md`
- **Prompt:** `prompt/formatting.prompt.md`
- **Use for:** Running formatters/linters before submitting changes.

## Logging & Commit

- **Guide:** `prompt/codex-logging-guide.md`
- **Prompt:** `prompt/logging.prompt.md`
- **Use for:** Writing timestamped logs and reusing them as `datalad` commit messages.

## Testing

- **Guide:** `prompt/codex-testing-guide.md`
- **Prompt:** `prompt/testing.prompt.md`
- **Use for:** Executing the project’s test suite with precise steps.

## Release Notes

- **Guide:** `prompt/codex-release-guide.md`
- **Prompt:** `prompt/release.prompt.md`
- **Use for:** Updating changelog entries during a release.
